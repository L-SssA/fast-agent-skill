# Feedback-Skill 使用指南

## 概述

`feedback-skill` 是一个用于从聊天上下文中提取用户对技能的纠正反馈的技能。它能够识别技能缺陷，生成规范化改进建议，并将这些洞察反馈到对应的技能包中，形成技能持续改进的闭环。

## 核心功能

- **上下文分析**：扫描对话历史，识别技能调用点和用户纠正行为
- **纠正检测**：识别显式和隐式的用户纠正信号
- **归因分析**：判断纠正是否由技能不完善导致
- **反馈提取**：从纠正中提取可规范化的具体问题和解决方案
- **智能整合**：将确认的改进点应用到对应技能中

## 前置条件

使用本技能前，确保：

- 存在包含技能使用过程的完整聊天上下文
- 用户在任务执行过程中有过纠正行为或明确反馈
- 能够识别出涉及的具体技能名称
- 目标技能包已存在且可以正常访问

## 使用流程

### 1. 分析聊天上下文

扫描对话历史，识别通过技能完成的任务和用户的纠正行为。

### 2. 检测纠正行为

查找用户纠正 agent 的交互片段，包括：

- 显式纠正："不对"、"应该是"、"你理解错了"等
- 隐式纠正：用户重新描述需求、提供修正示例
- 不满意信号：重复询问、表达困惑、要求重试

### 3. 归因分析

判断纠正是否源于：

- 技能描述不清
- 指令缺失或不准确
- 缺少边界情况说明
- 示例不足或误导

排除非技能因素（用户输入模糊、外部环境变化等）。

### 4. 提取改进点

从纠正中提取具体的问题描述和改进方向，按类型分类：

- 描述优化
- 指令补充
- 边界情况
- 示例增强

### 5. 生成分类报告

按问题类型组织改进建议，生成结构化的反馈报告。

### 6. 交互式确认

向用户展示提取结果，获得采纳确认。**始终进行交互式确认，不要自动应用未经用户确认的改进。**

### 7. 应用反馈

将确认的改进点整合到对应技能中，遵循渐进式披露原则。

### 8. 验证改进

建议用户使用 optimize-skill 测试验证改进效果。

## 典型应用场景

### 场景 1：用户纠正了技能的理解偏差

**问题**：用户使用技能时，agent 误解了任务意图，用户进行了纠正

**解决方案**：

1. 分析对话，定位纠正点
2. 进行归因分析
3. 判断是否因技能描述不清导致误解
4. 提取改进建议：优化 description 或添加澄清说明
5. 用户确认后，应用到技能

### 场景 2：技能遗漏了重要边界情况

**问题**：技能在特定边界条件下表现不佳，用户指出了这个问题

**解决方案**：

1. 识别边界情况的描述和处理方式
2. 提取规范化建议
3. 创建或更新 references/ 中的边界情况文档
4. 在 SKILL.md 中添加对边界的引用

### 场景 3：技能缺少关键步骤或指令

**问题**：agent 执行任务时遗漏了重要步骤，用户补充了指令

**解决方案**：

1. 提取用户补充的关键步骤
2. 判断应添加到 SKILL.md 的工作流还是 references/ 的详细指南
3. 更新相应的指令部分
4. 确保步骤顺序和逻辑清晰

### 场景 4：技能示例不够充分或存在误导

**问题**：技能提供的示例不足以覆盖实际场景，或有误导性

**解决方案**：

1. 收集用户提供的正确示例或指出示例问题
2. 创建新的示例文档或更新现有示例
3. 确保示例覆盖典型场景和边界情况
4. 在 SKILL.md 中更新示例引用

## 工作流程示例

```
用户: "刚才用 csv-analyzer 技能处理数据时，我纠正了它三次，感觉这个技能有问题"

你:
1. 确认要分析的技能是 "csv-analyzer"
2. 扫描最近的聊天上下文，找到使用该技能的对话
3. 进行分析，识别出三次纠正：
   - 纠正 1: agent 错误理解了日期格式 → 技能缺少格式说明
   - 纠正 2: agent 遗漏了空值处理 → 技能未提及边界情况
   - 纠正 3: agent 输出格式不符合要求 → 技能示例不清晰
4. 提取改进建议：
   - 【描述优化】在 description 中添加日期格式处理能力说明
   - 【边界情况】创建 data-validation.md 说明空值处理
   - 【示例增强】更新 output-format 示例，展示正确的输出格式
5. 向用户展示报告，请求确认
6. 用户确认所有改进点
7. 应用改进：
   - 优化 SKILL.md 的 description
   - 创建 data-validation.md
   - 更新示例章节
8. 生成应用报告
9. 建议使用 optimize-skill 进行测试验证
```

## 与相关技能的关系

- **create-skill**：从零创建新技能的基础框架
- **optimize-skill**：系统化优化技能质量、描述准确性和性能
- **enhance-skill**：从外部资料中提取信息并增强技能内容
- **feedback-skill**：从实际使用中提取用户反馈，持续改进技能

**典型工作流**：

1. 使用 create-skill 创建基础框架
2. 在实际使用中收集用户反馈
3. 使用 feedback-skill 从对话中提取改进点
4. 应用反馈到技能中
5. 使用 optimize-skill 进行系统化测试验证
6. 根据需要重复步骤 2-5 持续迭代

四个技能协同工作，形成技能创建 → 使用 → 反馈 → 优化 → 验证的完整闭环。

## 重要提醒

- **始终进行交互式确认**：不要自动应用未经用户确认的改进
- **准确归因**：仔细区分技能缺陷和其他因素导致的纠正
- **保持具体性**：提取的改进点必须具体、可操作，避免泛泛而谈
- **遵循渐进式披露**：详细内容放入 references/，保持 SKILL.md 简洁
- **注意内容去重**：避免添加与现有内容重复的改进
- **维护引用关系**：新增 reference 文件时，确保在 SKILL.md 中正确引用
- **关注可测试性**：提取的改进点应该能够通过 optimize-skill 进行验证

记住：你的目标是帮助用户**从实际使用中系统化地提取反馈**，将用户的纠正转化为技能的持续改进，形成良性循环。
