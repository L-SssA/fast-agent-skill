# enhance-skill 使用指南

## 技能概述

**enhance-skill** 是一个用于从用户提供的资料中提取有用信息并智能整合到指定技能包中的辅助技能。

### 功能定位

- 从各种类型的输入资料中提取有价值的信息
- 通过交互式确认流程确保内容质量
- 智能选择最佳的整合方式（更新文件/创建新文件/优化结构）
- 保持技能的清晰结构和职责边界

### 适用场景

- 有新技术文档需要整合到现有技能
- 从项目经验中提取最佳实践并固化到技能中
- 补充边界情况和注意事项
- 基于社区反馈改进技能内容
- 将外部知识系统化地添加到技能包

### 核心能力

- **多类型资料处理**：支持文档、代码、技术规范、对话记录等多种输入
- **智能信息提取**：自动识别与目标技能相关的有价值信息
- **交互式确认流程**：向用户展示提取的内容并获得确认
- **智能整合策略**：根据信息类型选择最优的整合方式
- **技能能力分析**：评估目标技能的当前能力和知识缺口
- **增强报告生成**：提供标准化的增强过程记录

## 使用方法

### 基本用法

当您有外部资料需要整合到现有技能时，向智能体描述需求：

```markdown
用户: "我有一份 Python 异步编程最佳实践的文档，想用来增强我的 python-async 技能"

智能体自动激活 enhance-skill，引导完成：

1. 确认目标技能 - 了解要增强哪个技能包
2. 分析技能能力 - 读取目标技能的当前状态
3. 接收输入资料 - 接受用户提供的各种资料
4. 智能提取信息 - 分析并提取相关信息
5. 交互式确认 - 展示提取内容并获得确认
6. 智能整合优化 - 执行整合并生成报告
```

### 支持的资料类型

enhance-skill 可以处理多种类型的输入资料：

#### 技术文档/API 文档

**特征**：结构化的技术规范、API 说明、参数列表

**提取重点**：

- 新的功能特性
- 使用方法的变化
- 参数配置的推荐值
- 性能优化建议
- 弃用警告和迁移指南

**示例**：

```markdown
用户: [粘贴 MySQL 8.0 新特性文档]

智能体提取：

- [NEW] CTE（公用表表达式）支持
- [BEST_PRACTICE] JSON 函数性能优化
- [GOTCHA] 默认字符集变更注意事项
```

#### 代码示例

**特征**：实际的代码实现、配置示例

**提取重点**：

- 设计模式和架构决策
- 错误处理策略
- 性能优化技巧
- 代码组织和模块化方法

**示例**：

```markdown
用户: [粘贴一段包含重试逻辑的代码]

智能体提取：

- [EXAMPLE] 指数退避重试模式
- [BEST_PRACTICE] 最大重试次数：3 次
- [EDGE_CASE] 仅对临时错误进行重试
```

#### 博客文章/教程

**特征**：叙述性的教学内容、经验分享

**提取重点**：

- 核心概念和原理
- 分步操作流程
- 常见问题和解决方案
- 作者的经验教训

**注意**：去除叙述性内容，只保留可操作的知识。

#### 对话记录/Q&A

**特征**：问答形式、问题排查过程

**提取重点**：

- 典型问题场景
- 诊断步骤
- 解决方案
- 根本原因分析
- 预防措施

#### 规范/标准文档

**特征**：行业标准、最佳实践指南

**提取重点**：

- 强制性要求
- 推荐做法
- 安全考虑
- 兼容性要求

### 关键参数

在使用 enhance-skill 时，准备好以下信息可以帮助智能体更好地协助您：

- **目标技能名称**：明确要增强哪个技能包
- **输入资料来源**：提供完整的资料内容或链接
- **增强目标**：希望解决什么问题或添加什么内容
- **优先级偏好**：是否有特别关注的内容类型

## 与其他技能的关系

### create-skill vs enhance-skill vs optimize-skill

三个技能各有不同的职责，协同工作：

| 技能               | 职责                   | 使用时机             | 频率     |
| ------------------ | ---------------------- | -------------------- | -------- |
| **create-skill**   | 从零创建技能的基础框架 | 新建技能时           | 一次     |
| **enhance-skill**  | 从外部资料增强技能内容 | 有新的知识需要整合时 | 按需多次 |
| **optimize-skill** | 优化技能质量和触发率   | 需要提升性能时       | 多次迭代 |

### 典型工作流

```
需求分析
    ↓
create-skill（创建基础框架）
    ↓
enhance-skill（从外部资料增强）←┐
    ↓                            │
optimize-skill（描述优化）        │ 持续改进
    ↓                            │
optimize-skill（质量提升）        │
    ↓                            │
optimize-skill（测试验证）←──────┘
    ↓
部署使用
    ↓
收集反馈
    ├→ enhance-skill（添加新知识）
    └→ optimize-skill（优化质量）
```

## 最佳实践

### 1. 明确增强目标

在使用 enhance-skill 之前，明确您希望通过增强达到什么目标：

- 添加新功能支持？
- 补充最佳实践？
- 修复已知问题？
- 更新过时内容？

### 2. 准备高质量的输入资料

输入资料的质量直接影响增强效果：

- 确保资料准确可靠
- 优先选择官方文档或权威来源
- 清理无关内容，聚焦核心信息
- 如有可能，提供结构化的资料

### 3. 仔细审查提取结果

虽然智能体会自动提取信息，但仍需人工审查：

- 检查相关性评分是否合理
- 确认分类是否准确
- 验证内容的准确性
- 拒绝不相关或低价值的信息

### 4. 保持技能职责单一

增强时注意不要破坏技能的单一职责：

- 避免引入与核心功能无关的内容
- 如发现功能膨胀，考虑拆分技能
- 定期检查技能的范围和边界

### 5. 遵循渐进式披露

始终确保增强后的技能保持良好的结构：

- SKILL.md 保持在 500 行以内
- 详细内容放入 references/ 目录
- 在 SKILL.md 中提供清晰的引用指引

### 6. 进行测试验证

增强完成后，建议进行测试：

- 使用 optimize-skill 评估触发率
- 验证新增内容的准确性
- 收集用户反馈
- 根据需要进一步调整

## 常见问题

### Q1: enhance-skill 会自动修改我的技能吗？

**A**: 不会。enhance-skill 采用交互式确认流程：

1. 首先提取信息并展示给您
2. 等待您确认哪些信息需要添加
3. 制定整合方案并再次确认
4. 获得您的最终批准后才执行

整个过程您都有完全的控制权。

### Q2: 如何处理大量资料？

**A**: enhance-skill 可以处理大量资料：

- 如果资料很长，可以分批提供
- 智能体会自动筛选最相关的内容
- 您可以指定重点关注的内容类型
- 建议先提供最重要的部分

### Q3: 增强后发现内容重复怎么办？

**A**: enhance-skill 会在提取阶段检测重复：

- 完全重复的内容会被标记并跳过
- 部分重叠的内容会建议合并
- 您可以在确认阶段拒绝重复内容
- 如仍有问题，可以使用 optimize-skill 进行清理

### Q4: 增强会影响技能的触发率吗？

**A**: 可能的影响：

- **正面影响**：添加更多关键词和场景描述可能提高触发率
- **负面影响**：如添加过多不相关内容可能降低准确性
- **建议**：增强后使用 optimize-skill 测试触发率，必要时调整 description

### Q5: 可以撤销增强操作吗？

**A**: 建议的做法：

- 在增强前备份技能文件
- 使用 Git 进行版本控制
- enhance-skill 会生成增强报告，记录所有变更
- 如需撤销，可以从备份恢复或使用 Git 回滚

### Q6: 多久应该增强一次技能？

**A**: 没有固定频率，取决于：

- 相关技术的更新速度
- 新知识的积累情况
- 用户反馈的需求
- 建议：当有重要新知识时及时增强，避免积压

### Q7: enhance-skill 能处理代码示例吗？

**A**: 可以。enhance-skill 特别擅长处理代码：

- 提取设计模式和最佳实践
- 识别错误处理策略
- 分析性能优化技巧
- 生成可复用的代码片段
- 添加必要的注释和说明

### Q8: 如何评估增强的效果？

**A**: 可以从以下几个方面评估：

- **内容完整性**：是否覆盖了重要的知识点
- **结构清晰度**：是否保持了良好的组织
- **可操作性**：新增内容是否可以直接应用
- **用户反馈**：实际使用者的评价
- **触发率变化**：使用 optimize-skill 测试
- **输出质量**：技能产生的结果是否更好
