# optimize-skill 使用指南

## 技能概述

**optimize-skill** 是一个用于优化和改进现有 Agent Skills 质量的辅助技能。

### 功能定位

- 提升技能的触发准确性和输出质量
- 应用经过验证的最佳实践
- 提供系统化的测试和评估方法

### 适用场景

- 已有基础技能框架需要优化
- 技能触发率低，很少被激活
- 输出结果不稳定或不准确
- 需要进行系统化测试验证

### 核心能力

- **描述优化**：系统化提升 description 字段的触发准确性
- **最佳实践应用**：应用 Gotchas、模板、清单等有效指令模式
- **评估测试**：触发率评估、输出质量评分、自动化验证
- **持续改进**：基于反馈迭代，维护测试用例集

## 使用方法

### 基本用法

当您遇到技能质量问题时，向智能体描述具体问题：

```markdown
用户: "我的 CSV 分析技能触发率很低"

智能体自动激活 optimize-skill，引导完成：

1. 问题诊断 - 了解当前状况和具体表现
2. 选择优化方向 - 描述优化/最佳实践/测试验证
3. 执行优化流程 - 加载对应参考文档，逐步改进
4. 测试验证 - 评估优化效果，持续迭代
```

### 优化方向

optimize-skill 支持三个主要优化方向：

#### 1. 描述优化

**参考文档**：[description-optimization.md](../skills/optimize-skill/references/description-optimization.md)

适用于：

- 技能很少被激活
- 技能在不相关场景下误触发
- 需要提升触发准确性

包含内容：

- 描述优化策略（祈使句、关注用户意图、覆盖边界情况）
- 评估查询集设计（应触发/不应触发）
- 训练/验证集划分避免过拟合
- 优化循环：评估 → 识别失败 → 修订 → 重复
- 自动化脚本模板

#### 2. 最佳实践应用

**参考文档**：[best-practices.md](../skills/optimize-skill/references/best-practices.md)

适用于：

- 输出质量不稳定
- 代理执行效率低
- 需要提升指令清晰度

包含内容：

- 从真实经验出发提取模式
- 上下文管理（添加代理缺乏的，省略已知的）
- 控制级别校准（自由度 vs 规定性）
- 有效指令模式（Gotchas、模板、清单、验证循环等）

#### 3. 测试验证

**参考文档**：[evaluation-testing.md](../skills/optimize-skill/references/evaluation-testing.md)

适用于：

- 需要量化评估技能性能
- 建立系统化测试流程
- 生成评估报告

包含内容：

- 触发率评估：多次运行计算触发率
- 输出质量评估：设计测试用例、定义评分标准
- 自动化验证脚本框架
- 综合评估报告生成

### 交互示例

```markdown
用户: "我的 PDF 处理技能输出质量不稳定，有时能正确提取文本，有时会遗漏内容"

智能体:

3. 加载 references/best-practices.md，开始优化：
   - 识别缺失的关键信息

   "已应用最佳实践改进指令。建议使用 evaluation-testing.md

```

## 典型应用场景

### 场景 1：技能触发率低

**问题表现**：

- 技能很少被代理激活
- 相关查询没有触发技能

**解决方案**：

1. 加载 [description-optimization.md](../skills/optimize-skill/references/description-optimization.md)
2. 设计评估查询集：
   - 8-10 个应触发的查询（措辞变化、显式程度、细节量、复杂度）
   - 8-10 个不应触发的查询（相似但不相关的任务）
3. 执行优化循环：
   - 测试当前描述的触发情况
   - 识别失败的查询
   - 修订 description（扩大范围、添加关键词、澄清边界）
   - 重复测试直到满意
4. 使用训练/验证集划分避免过拟合

**预期效果**：触发率显著提升，相关查询能稳定激活技能

---

### 场景 2：技能误触发

**问题表现**：

- 技能在不相关场景下被激活
- 干扰其他技能的使用

**解决方案**：

1. 加载 [description-optimization.md](../skills/optimize-skill/references/description-optimization.md)
2. 识别误触发的查询模式
3. 增加描述特异性：
   - 明确技能边界
   - 添加排除条件
   - 澄清不适用的场景
4. 测试验证不再误触发

**预期效果**：误触发率降低，技能只在相关场景激活

---

### 场景 3：输出质量不稳定

**问题表现**：

- 技能产生的结果不一致
- 有时准确，有时遗漏关键信息
- 输出格式不统一

**解决方案**：

1. 加载 [best-practices.md](../skills/optimize-skill/references/best-practices.md)
2. 应用有效指令模式：
   - **添加 Gotchas 章节**：列出违反直觉的事实和边界情况
   - **提供输出模板**：明确的格式要求和示例
   - **添加验证循环**：确保完整性和准确性
   - **使用清单**：分步检查关键点
3. 提供默认方案而非多个选项
4. 删除代理已知的冗余内容

**预期效果**：输出质量稳定，格式统一，关键信息不遗漏

---

### 场景 4：技能效率低

**问题表现**：

- 代理执行任务时步骤冗余
- 花费过多时间在无成效的步骤上
- 选择了次优的方法

**解决方案**：

1. 加载 [best-practices.md](../skills/optimize-skill/references/best-practices.md)
2. 优化策略：
   - **提供默认方案**：选择一个推荐方法，减少决策时间
   - **捆绑可复用脚本**：将复杂操作封装到 scripts/
   - **优化指令**：减少不必要的步骤
   - **校准控制级别**：根据任务复杂度调整规定性
3. 阅读代理执行轨迹，识别浪费时间的步骤
4. 精简指令，聚焦关键操作

**预期效果**：执行效率提升，步骤更简洁，更快完成任务

## 与 create-skill 的关系

### 职责分工

- **create-skill**：专注于从零创建符合标准的基础框架
  - 确保目录结构正确
  - 编写规范的 frontmatter
  - 搭建基础指令框架
- **optimize-skill**：专注于优化和改进现有技能的质量
  - 提升描述准确性
  - 应用最佳实践
  - 系统化测试验证

### 典型工作流

```
1. 使用 create-skill 创建基础框架
       ↓
2. 使用 optimize-skill 优化描述和质量
       ↓
3. 使用 optimize-skill 进行测试验证
       ↓
4. 根据测试结果迭代改进
       ↓
5. 部署到生产环境，收集反馈
       ↓
6. 持续优化（回到步骤 2）
```

两个技能协同工作，覆盖技能的完整生命周期。

## 常见问题

### Q: 什么时候应该使用 optimize-skill？

**A**: 当您已有基础技能框架但遇到以下问题时：

- 技能触发率低或误触发
- 输出质量不稳定
- 需要应用最佳实践
- 需要进行系统化测试

如果是从零创建技能，请先使用 create-skill。

### Q: 如何进行系统化测试？

**A**: 使用 [evaluation-testing.md](../skills/optimize-skill/references/evaluation-testing.md) 中的方法：

1. **触发率测试**：
   - 准备 20 个测试查询（10 个应触发，10 个不应触发）
   - 多次运行（建议 5-10 次）
   - 计算触发率 = 正确触发次数 / 总运行次数

2. **输出质量评估**：
   - 设计测试用例覆盖不同场景
   - 定义评分标准（准确性、完整性、格式等）
   - 人工或自动化评分

3. **使用评估脚本**：
   ```bash
   # 使用提供的模板
   cp assets/eval-script-template.sh eval-my-skill.sh
   # 自定义测试用例
   # 运行评估
   bash eval-my-skill.sh
   ```

### Q: 优化需要多少轮迭代？

**A**: 取决于问题的复杂程度：

- **简单优化**（如调整描述）：3-5 轮迭代通常足够
- **中等复杂度**（应用最佳实践）：5-10 轮迭代
- **复杂优化**（全面提升质量）：可能需要 10+ 轮迭代

**关键原则**：

- 每轮迭代后都要测试验证
- 使用训练/验证集划分避免过拟合
- 当改进趋于平缓时可以停止
- 持续收集实际使用反馈

### Q: 如何判断优化是否成功？

**A**: 通过量化指标和主观评估结合：

**量化指标**：

- 触发率提升（目标 > 80%）
- 误触发率降低（目标 < 10%）
- 输出质量评分提升

**主观评估**：

- 代理执行更高效
- 输出更稳定和一致
- 用户反馈更积极

### Q: 优化会影响已有的功能吗？

**A**: 有可能，因此需要：

1. **保留备份**：优化前备份原始版本
2. **回归测试**：确保原有功能仍然正常
3. **渐进式改进**：每次只改动一小部分
4. **充分测试**：每轮迭代后都进行测试验证

使用版本控制（如 Git）可以更好地跟踪变化和回滚。

## 最佳实践提示

1. **系统化方法**：不要凭感觉优化，使用评估查询集和数据驱动
2. **小步快跑**：每次只做少量改动，便于定位问题
3. **避免过拟合**：使用训练/验证集划分，确保泛化能力
4. **记录变化**：跟踪每次优化的内容和效果
5. **持续迭代**：优化是持续过程，不是一次性任务
6. **结合实际反馈**：除了测试集，还要关注真实用户的反馈
