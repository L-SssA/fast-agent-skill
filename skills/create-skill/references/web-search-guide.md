# 网络搜索增强指南

本指南提供在用户信息不完整时，通过网络搜索获取补充资料的系统化方法。帮助你在创建技能的过程中主动收集必要的背景知识、技术规范和最佳实践。

## 核心原则

### 1. 用户授权优先

**始终在进行搜索前征得用户同意**：

- 明确说明为什么要搜索
- 告知将要搜索的内容范围
- 让用户确认或调整搜索方向
- 尊重用户拒绝搜索的决定

**示例对话**：

```
我注意到您提供的信息中缺少关于 [具体缺失点] 的详细说明。
为了创建更准确的技能，我可以搜索相关的技术文档和最佳实践。

建议搜索：
- [关键词1]
- [关键词2]

是否允许我进行网络搜索？或者您有其他偏好？
```

### 2. 目标导向搜索

搜索应该有明确的目标，避免漫无目的的浏览：

✓ 填补具体的知识缺口  
✓ 验证技术细节的准确性  
✓ 获取最新的 API 文档或规范  
✓ 查找官方推荐的最佳实践  
✗ 泛泛地了解某个主题  
✗ 收集大量不相关的背景信息

### 3. 来源可靠性评估

优先使用权威和可靠的来源：

**高可信度来源**：

- 官方文档（python.org, docs.microsoft.com 等）
- 官方 GitHub 仓库和 README
- 知名的技术博客（公司官方博客、知名开发者博客）
- 经过验证的 Stack Overflow 答案（高票、已接受）
- 学术论文和技术报告

**中等可信度来源**：

- 社区教程和指南
- 技术论坛讨论
- Medium、Dev.to 等平台的文章

**低可信度来源**（需谨慎验证）：

- 个人博客（无权威背书）
- 过时的文章（超过 2-3 年）
- 未经验证的说法

### 4. 信息时效性

技术信息更新迅速，注意：

- 优先查找最新版本的信息
- 标注信息的版本适用范围
- 警惕过时的 API 或已弃用的功能
- 如有版本冲突，以官方最新文档为准

## 何时使用网络搜索

### 应该搜索的情况

1. **用户明确表示不了解**
   - "我不太清楚这个技术的具体用法"
   - "你能帮我查一下最新的做法吗？"

2. **技术领域快速变化**
   - 新的框架版本发布
   - API 有重大更新
   - 最佳实践发生变化

3. **需要验证关键信息**
   - 用户对某个细节不确定
   - 存在多个相互矛盾的说法
   - 需要官方文档确认

4. **缺少具体的技术参数**
   - 默认配置值
   - 性能指标
   - 兼容性要求

5. **需要实际示例代码**
   - 官方示例
   - 社区验证的实现
   - 完整的工作流演示

### 不应搜索的情况

1. **用户已提供充分信息**
   - 需求明确且完整
   - 已有足够的技术细节

2. **基础知识**
   - 代理已经掌握的通用知识
   - 标准的编程概念

3. **项目特定的内容**
   - 内部业务逻辑
   - 私有 API 或系统

4. **用户明确要求不搜索**
   - 尊重用户的偏好

## 搜索策略

### Step 1: 识别信息缺口

分析用户提供的需求，识别缺失的关键信息：

**检查清单**：

- [ ] 技能的核心功能是否明确？
- [ ] 使用的技术栈是否清晰？
- [ ] 输入输出格式是否定义？
- [ ] 边界情况是否考虑？
- [ ] 是否有版本要求？
- [ ] 是否需要特定的依赖？

**示例**：

```
用户需求："创建一个处理 Excel 文件的技能"

识别的缺口：
- 使用哪个库？(openpyxl, pandas, xlrd?)
- 处理什么类型的操作？(读取、写入、格式化？)
- 支持的 Excel 版本？(.xlsx, .xls?)
- 性能要求？(大文件处理？)
- 错误处理策略？
```

### Step 2: 构建搜索查询

设计精准的搜索查询语句：

**技巧**：

1. **使用具体关键词**
   - ✓ `python openpyxl read excel best practices 2024`
   - ✗ `excel python`

2. **包含版本信息**
   - ✓ `pandas read_excel parameters v2.0`
   - ✗ `pandas read excel`

3. **指定信息来源**
   - ✓ `site:docs.python.org asyncio timeout`
   - ✓ `site:github.com microsoft playwright examples`

4. **使用问题形式**
   - ✓ `how to handle large excel files in python memory efficient`
   - ✗ `large excel files`

5. **组合多个关键词**
   - ✓ `react hooks useEffect dependency array common mistakes`
   - ✗ `react hooks`

### Step 3: 执行搜索

使用可用的搜索工具：

**推荐搜索顺序**：

1. **官方文档**（最高优先级）
   - 直接访问项目的官方文档网站
   - 查找 API 参考、快速开始、最佳实践章节

2. **GitHub 仓库**
   - 查看 README.md
   - 浏览 examples/ 目录
   - 检查 Issues 中的常见问题
   - 查看最近的 releases 和 changelog

3. **技术社区**
   - Stack Overflow（按票数排序）
   - Reddit 相关 subreddit
   - 专业技术论坛

4. **技术博客和教程**
   - 公司官方博客
   - 知名开发者博客
   - 高质量的技术媒体

### Step 4: 筛选和验证

对搜索结果进行评估：

**评估标准**：

1. **相关性**
   - 是否直接回答了我的问题？
   - 是否适用于当前场景？

2. **权威性**
   - 来源是否可靠？
   - 作者是否有资质？

3. **时效性**
   - 信息是否最新？
   - 是否适用于当前版本？

4. **完整性**
   - 是否提供了足够的细节？
   - 是否有示例代码？

5. **一致性**
   - 与其他可靠来源是否一致？
   - 是否有矛盾之处？

### Step 5: 提取和组织信息

从搜索结果中提取有价值的内容：

**提取要点**：

- 关键的 API 用法和参数
- 推荐的配置和默认值
- 常见的陷阱和注意事项
- 完整的代码示例
- 性能优化建议
- 版本兼容性信息

**组织方式**：

```markdown
## 搜索结果摘要

### 来源 1: [标题](URL)

- 关键点 1
- 关键点 2
- 适用版本：X.Y+

### 来源 2: [标题](URL)

- 关键点 1
- 关键点 2

### 综合建议

基于以上信息，建议采用...
```

## 与用户交互的流程

### 流程 1: 请求搜索许可

```markdown
## 信息补充建议

我注意到以下信息对于创建高质量的技能很重要，但当前还不明确：

**缺失的信息**：

1. [具体缺失点 1]
2. [具体缺失点 2]

**建议的搜索方向**：

- 搜索 [关键词 1] 了解 [目的]
- 搜索 [关键词 2] 获取 [目的]

**预期收益**：

- 更准确的技术选型
- 最新的最佳实践
- 完整的示例代码

是否允许我进行网络搜索来补充这些信息？

选项：

1. ✅ 是的，请搜索所有建议的内容
2. 🔍 只搜索特定的部分：[用户指定]
3. ❌ 不需要，我提供更多信息
4. 💬 先讨论一下搜索策略
```

```markdown

我进行了以下搜索并找到了有价值的信息：

### 搜索 1: [查询语句]

- 发现 1：详细说明
- 发现 2：详细说明
- 代码示例：[如果有]

**适用性**：这解决了 [具体问题]

---

### 搜索 2: [查询语句]

**来源**: [链接标题](URL)

**关键发现**：

- ...

---

## 建议整合方案

基于搜索结果，我建议：

1. 采用 [技术方案 A]，因为 [理由]
2. 在技能中包含 [具体内容]
3. 添加注意事项：[注意点]

您希望：

1. ✅ 使用这些信息进行技能创建
2. ✏️ 调整某些部分后再继续
3. 🔍 搜索更多相关信息
4. ❌ 不使用搜索结果，采用其他方式
```

### 流程 3: 处理冲突信息

当不同来源提供矛盾信息时：

```markdown
## 检测到信息冲突

关于 [具体问题]，我发现了不同的说法：

**来源 A** ([链接]):

- 说法 1
- 版本：X.Y
- 日期：2024-01

**来源 B** ([链接]):

- 说法 2
- 版本：X.Z
- 日期：2024-03

**分析**：

- 来源 B 更新，可能反映了最新的变更
- 来源 A 是官方文档，权威性更高
- 可能适用于不同的使用场景

**建议**：
采用来源 B 的方法，因为它是最新的，并且...

您希望：

1. ✅ 采用建议的方案
2. 📖 查看更多来源
3. 🤔 采用保守的方案（来源 A）
4. 💬 讨论其他选项
```

## 搜索技巧和最佳实践

### 高级搜索运算符

**限定网站**：

```
site:docs.python.org async await
site:github.com facebook/react hooks
```

**精确匹配**：

```
"useEffect dependency array"
"memory leak prevention"
```

**排除词汇**：

```
python async -tutorial -beginner
```

**文件类型**：

```
filetype:pdf kubernetes security best practices
```

**时间范围**：

```
react server components after:2023
```

### 多语言搜索

如果英文资源不足，可以尝试：

- 中文：适合国内特定技术生态
- 日文：某些日本主导的技术
- 其他语言：根据技术的起源地区

**注意**：优先使用英文，因为大多数技术的官方文档和社区讨论都是英文。

### 交叉验证

对于关键信息，至少从 2-3 个独立来源验证：

1. 官方文档
2. GitHub 示例
3. 社区验证（Stack Overflow 高票答案）

如果三个来源一致，可信度高。

### 记录搜索过程

保留搜索记录便于追溯：

```markdown
## 搜索记录

**日期**: 2026-04-18

**搜索 1**:

- 查询: `python asyncio timeout best practices`
- 来源: https://docs.python.org/3/library/asyncio-task.html
- 关键信息: timeout 参数使用方法
- 采用: 是

**搜索 2**:

- 查询: `openpyxl large file memory optimization`
- 来源: https://openpyxl.readthedocs.io/en/stable/optimized.html
- 关键信息: 使用 read_only 模式处理大文件
- 采用: 是
```

## 注意事项和陷阱

### Gotcha 1: 过时信息

**问题**：搜索到的信息已过时，不适用于当前版本

**避免方法**：

- 始终检查发布日期
- 验证适用的版本范围
- 优先选择最近 1-2 年的内容
- 对照官方文档确认

### Gotcha 2: 上下文不匹配

**问题**：找到的解决方案适用于不同的场景

**避免方法**：

- 仔细阅读完整篇文章，理解上下文
- 确认是否适用于你的具体情况
- 寻找多个来源验证适用性

### Gotcha 3: 过度依赖单一来源

**问题**：只参考了一个来源，可能存在偏差

**避免方法**：

- 至少查阅 2-3 个独立来源
- 比较不同观点
- 优先选择有多个来源支持的说法

### Gotcha 4: 忽略许可证和版权

**问题**：使用了受版权保护的代码或内容

**避免方法**：

- 检查代码示例的许可证
- 引用来源时要注明
- 避免直接复制大段内容
- 用自己的话重述概念

### Gotcha 5: 搜索偏见

**问题**：只看到符合预期的结果

**避免方法**：

- 尝试不同的搜索词
- 阅读反对观点
- 保持开放心态
- 验证自己的假设

## 总结

网络搜索是增强技能创建能力的有力工具，但需要谨慎和系统化地使用。遵循本指南的原则和方法，可以：

1. **高效获取信息**：快速填补知识缺口
2. **保证质量**：使用可靠和最新的信息
3. **尊重用户**：始终获得授权并透明操作
4. **避免陷阱**：识别和处理常见问题

记住核心要点：

- 始终先征得用户同意
- 优先使用官方和权威来源
- 交叉验证关键信息
- 保持信息的时效性和相关性
- 清晰地与用户沟通搜索结果

通过负责任的网络搜索，你可以创建更加准确、全面和实用的技能。
