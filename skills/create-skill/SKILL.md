# Create Skill - 技能创建助手

本技能用于指导用户创建高质量、符合官方标准的 Agent Skills。

## 使用流程

1. **了解需求**: 询问用户要创建什么类型的技能，目标场景是什么
2. **评估信息完整性**: 检查用户提供的信息是否充分
   - 如果信息不完整 → 征求用户同意后使用网络搜索补充（参考 [web-search-guide.md](references/web-search-guide.md)）
   - 如果信息充足 → 继续下一步
3. **选择路径**:
   - 从零创建 → 参考 [framework.md](references/framework.md)
   - 需要模板 → 参考 [assets/skill-template.md](assets/skill-template.md)
4. **执行指导**: 加载对应的参考文档，按步骤引导用户完成基础框架搭建
5. **后续建议**: 完成后建议使用 optimize-skill 进行优化和验证

## 关键原则摘要

### 命名规范

- 仅允许小写字母、数字、连字符（`a-z`, `0-9`, `-`）
- 长度 1-64 字符
- 不能以连字符开头或结尾
- 不能有连续连字符（`--`）
- 必须与父目录名完全匹配

### 描述要点

- 长度 1-1024 字符
- 必须包含"做什么"和"何时使用"两部分
- 使用祈使句："Use this skill when..." 而非 "This skill does..."
- 包含具体关键词帮助代理识别相关任务

**注意**：以上为基础要求。如需高级优化技巧（如主动覆盖边界情况、系统化测试触发率等），请使用 optimize-skill 技能。

### 渐进式披露

- SKILL.md 保持在 500 行以内
- 详细内容放入 `references/` 目录
- 在 SKILL.md 中明确说明何时加载哪些参考文件
- 文件引用使用相对路径，避免深层嵌套

### 最佳实践核心（基础）

- **从真实经验出发**: 基于实际任务提取模式，而非通用知识
- **提供默认方案**: 选择一个默认方法，简要提及替代方案
- **聚焦代理不知道的内容**: 添加项目约定、领域特定程序

**注意**：更多高级最佳实践（如上下文管理、控制级别校准、有效指令模式等）请参考 optimize-skill 技能。

## 何时加载详细指南

根据用户需求判断阶段并加载对应文档：

- **从零创建技能** → 读取 [`references/framework.md`](references/framework.md)
  - 包含目录结构规范、frontmatter 详解、body 编写基础指南、实操示例

- **需要标准模板** → 参考 [`assets/skill-template.md`](assets/skill-template.md)
  - 提供完整的技能模板，包含所有必要章节

- **信息不完整需要补充** → 读取 [`references/web-search-guide.md`](references/web-search-guide.md)
  - 包含网络搜索策略、用户授权流程、来源可靠性评估、搜索技巧和最佳实践
  - 在用户同意后进行有针对性的信息收集

**重要提示**：技能创建完成后，如需优化描述、应用最佳实践或进行测试验证，请使用 **optimize-skill** 技能。

## 快速检查清单

在帮助用户完成技能创建后，确保：

- [ ] 目录名与 frontmatter 中的 `name` 字段一致
- [ ] `name` 符合命名规范（小写、连字符、长度限制）
- [ ] `description` 包含"做什么"和"何时使用"，不超过 1024 字符
- [ ] SKILL.md 行数 < 500
- [ ] 使用了渐进式披露（详细内容在 references/ 中）
- [ ] 文件引用使用相对路径
- [ ] 提供了清晰的步骤和示例
- [ ] 包含了边界情况和注意事项

## 常见错误提醒

1. **描述过于宽泛**: "Helps with PDFs" → 应改为具体说明功能和使用时机
2. **描述过于技术化**: 关注用户意图而非内部实现机制
3. **SKILL.md 过长**: 超过 500 行应考虑将内容移至 references/
4. **缺少触发关键词**: 描述中未包含用户可能使用的词汇
5. **命名不规范**: 使用了大写字母、空格或特殊字符

## 工作流程示例

### 示例 1: 标准信息创建

```
用户: "我想创建一个处理 CSV 数据的技能"

你:
1. 确认需求细节（数据类型、处理方式、输出格式等）
2. 加载 references/framework.md
3. 指导用户创建目录结构和 SKILL.md
4. 帮助编写 frontmatter 和基础 body 内容
5. 使用 scripts/validate-skill.py 验证技能是否符合规范
6. 完成后建议使用 optimize-skill 进行进一步优化和测试
```

### 示例 2: 信息不完整时使用网络搜索

```
用户: "我想创建一个使用最新 AI API 的技能，但不太清楚具体怎么用"

你:
1. 识别信息缺口：具体的 API 名称、版本、认证方式等
2. 向用户请求搜索许可：
   "我注意到缺少一些关键技术细节。为了创建更准确的技能，我可以搜索相关的官方文档和最佳实践。
   建议搜索：[AI API 名称] 官方文档、认证方法、使用示例。
   是否允许我进行网络搜索？"
3. 用户同意后，加载 references/web-search-guide.md
4. 执行有针对性的搜索，获取最新信息
5. 向用户展示搜索结果并确认
6. 基于收集的信息，继续技能创建流程
7. 使用 validate-skill.py 验证
8. 建议使用 optimize-skill 进行优化
```

## 验证脚本使用

本技能提供了自动化验证脚本，用于检查技能是否符合官方规范：

### 使用方法

```bash
# 验证当前目录的技能
uv run scripts/validate-skill.py .

# 验证指定目录的技能
uv run scripts/validate-skill.py /path/to/skill
```

### 验证内容

脚本会自动检查以下项目：

- ✅ Frontmatter 必要字段（name, description）
- ✅ 命名规范（小写、连字符、长度限制）
- ✅ Description 质量（长度、功能说明、使用时机、祈使句）
- ✅ SKILL.md 行数限制（< 500 行）
- ✅ 渐进式披露（references/ 目录使用）
- ✅ 文件引用路径（相对路径）
- ✅ 目录结构完整性

### 退出码

- `0`: 验证通过（无错误）
- `1`: 验证失败（存在错误）

记住：你的目标是帮助用户创建**符合标准的基础框架**。使用验证脚本确保质量，如需进一步提升，请引导用户使用 optimize-skill。
