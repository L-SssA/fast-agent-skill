# 信息提取指南

本指南提供从各种类型的输入资料中提取有价值信息的系统化方法，用于增强现有技能包。

## 核心原则

### 1. 相关性优先

提取的信息必须与目标技能的核心职责直接相关。问自己：

- 这条信息是否帮助用户更好地使用这个技能？
- 是否填补了技能当前的知识缺口？
- 是否与技能的现有内容形成互补而非重复？

### 2. 价值导向

优先提取高价值信息：

- **可操作的知识**：具体的步骤、方法、技巧
- **边界情况**：特殊场景的处理方法
- **最佳实践**：经过验证的有效模式
- **常见陷阱**：需要避免的错误和注意事项
- **示例代码/配置**：可直接复用的代码片段

### 3. 结构化思维

将提取的信息按类别组织：

- 新增知识（技能尚未覆盖的内容）
- 最佳实践（改进现有方法的建议）
- 边界情况（特殊场景处理）
- 示例和模板（可复用的代码或配置）
- 注意事项和陷阱（需要警惕的问题）
- 参考资料（外部资源链接）

## 不同类型资料的处理方法

### 技术文档/API 文档

**特征**：结构化的技术规范、API 说明、参数列表

**提取重点**：

- 新的功能特性
- 使用方法的变化
- 参数配置的推荐值
- 性能优化建议
- 弃用警告和迁移指南

**示例**：

```
原文: "从 v2.0 开始，async_connect() 方法支持 timeout 参数，推荐设置为 30 秒以避免长时间阻塞。"

提取为:
类别: 新增知识 + 最佳实践
内容:
  - async_connect() 新增 timeout 参数（v2.0+）
  - 推荐 timeout 值：30 秒
  - 用途：避免长时间阻塞
```

### 代码示例

**特征**：实际的代码实现、配置示例

**提取重点**：

- 设计模式和架构决策
- 错误处理策略
- 性能优化技巧
- 代码组织和模块化方法
- 注释中解释的关键逻辑

**示例**：

```
原文: [一段包含重试逻辑的代码]

提取为:
类别: 示例代码 + 最佳实践
内容:
  - 指数退避重试模式实现
  - 最大重试次数：3 次
  - 初始延迟：1 秒，倍增因子：2
  - 仅对临时错误进行重试
```

### 博客文章/教程

**特征**：叙述性的教学内容、经验分享

**提取重点**：

- 核心概念和原理
- 分步操作流程
- 常见问题和解决方案
- 作者的经验教训
- 推荐的工具和资源

**注意**：去除叙述性内容，只保留可操作的知识。

### 对话记录/Q&A

**特征**：问答形式、问题排查过程

**提取重点**：

- 典型问题场景
- 诊断步骤
- 解决方案
- 根本原因分析
- 预防措施

**示例**：

```
原文:
  Q: "为什么我的异步任务会阻塞主线程？"
  A: "因为你使用了 asyncio.run() 在已运行的事件循环中。应该使用 await 或 create_task()。"

提取为:
类别: 边界情况 + 注意事项
内容:
  - 问题：在已运行的事件循环中使用 asyncio.run() 会导致阻塞
  - 解决：使用 await 或 create_task()
  - 场景：嵌套异步调用时容易出现
```

### 规范/标准文档

**特征**：行业标准、最佳实践指南

**提取重点**：

- 强制性要求
- 推荐做法
- 安全考虑
- 兼容性要求
- 版本差异

## 相关性判断框架

使用以下标准判断信息是否应该提取：

### 高相关性（应该提取）

✓ 直接影响技能核心功能的正确使用  
✓ 解决技能当前未覆盖的常见场景  
✓ 提供比现有内容更优的方法  
✓ 补充重要的边界情况或异常处理  
✓ 包含可立即应用的示例或模板

### 中相关性（酌情提取）

~ 扩展技能的适用范围但非核心  
~ 提供替代方案但现有方案已足够  
~ 理论背景知识但不影响实际操作  
~ 高级用法但大多数用户不需要

### 低相关性（不应提取）

✗ 与技能职责无关的内容  
✗ 过于基础的知识（代理已掌握）  
✗ 过时的信息或已弃用的方法  
✗ 过于特定于某个项目的细节  
✗ 纯理论而无实际应用价值

## 信息分类体系

### 1. 新增知识 (New Knowledge)

技能完全未覆盖的新内容。

**标识**：`[NEW]`  
**整合方式**：通常创建新的 reference 文件或添加到现有文件的适当章节

**示例**：

```
[NEW] Python 3.12 新增的 exception_group 特性
- 用于同时处理多个异常
- 使用 except* 语法捕获
- 适用于并行任务中的错误处理
```

### 2. 最佳实践 (Best Practices)

改进现有方法的建议。

**标识**：`[BEST_PRACTICE]`  
**整合方式**：更新现有 reference 文件或在 SKILL.md 中添加引用

**示例**：

```
[BEST_PRACTICE] 数据库连接池配置
- 最小连接数：5
- 最大连接数：20
- 空闲超时：300 秒
- 基于实际负载测试得出
```

### 3. 边界情况 (Edge Cases)

特殊场景或异常情况的处理方法。

**标识**：`[EDGE_CASE]`  
**整合方式**：添加到 Gotchas 章节或创建专门的边界情况文档

**示例**：

```
[EDGE_CASE] 空列表处理
- map() 对空列表返回空迭代器
- 不会执行任何转换
- 需要显式检查 if not list: 来处理特殊情况
```

### 4. 示例和模板 (Examples & Templates)

可直接复用的代码、配置或模板。

**标识**：`[EXAMPLE]`  
**整合方式**：创建 examples/ 目录或添加到现有文件的示例章节

**示例**：

```
[EXAMPLE] REST API 错误响应格式
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input",
    "details": [...]
  }
}
```

### 5. 注意事项和陷阱 (Gotchas & Warnings)

需要特别警惕的问题和常见错误。

**标识**：`[GOTCHA]` 或 `[WARNING]`  
**整合方式**：添加到 Gotchas 章节，使用醒目的格式

**示例**：

```
[GOTCHA] Python 可变默认参数
- 不要在函数定义中使用可变对象作为默认参数
- 错误: def func(items=[])
- 正确: def func(items=None): items = items or []
- 原因：默认参数在函数定义时只求值一次
```

### 6. 参考资料 (References)

外部资源、文档链接、进一步阅读材料。

**标识**：`[REFERENCE]`  
**整合方式**：添加到 References 章节或相关内容的末尾

**示例**：

```
[REFERENCE]
- 官方文档: https://docs.python.org/3/library/asyncio.html
- PEP 492: Coroutines with async and await syntax
- 深入理解: "Fluent Python" 第 18 章
```

## 提取质量评估标准

在完成信息提取后，使用以下标准评估质量：

### 完整性

- [ ] 提取了所有高相关性的信息
- [ ] 保留了关键的上下文信息
- [ ] 包含了必要的示例或代码
- [ ] 标注了版本要求或依赖条件

### 准确性

- [ ] 信息准确无误，没有曲解原意
- [ ] 代码示例可以正常运行
- [ ] 引用了正确的来源
- [ ] 标注了不确定或需要验证的内容

### 可操作性

- [ ] 信息可以直接应用于技能
- [ ] 提供了清晰的步骤或指导
- [ ] 示例具有代表性
- [ ] 说明了适用场景和限制

### 结构化

- [ ] 信息按类别清晰组织
- [ ] 使用了统一的标识格式
- [ ] 层次分明，易于理解
- [ ] 便于后续整合到技能中

### 简洁性

- [ ] 去除了冗余和重复内容
- [ ] 语言精炼，没有废话
- [ ] 聚焦关键点
- [ ] 避免了过度详细的背景说明

## 提取工作流程

### Step 1: 快速浏览

快速扫描整个资料，了解：

- 主要主题和结构
- 关键章节和内容
- 与目标技能的相关程度

### Step 2: 标记候选信息

逐段阅读，标记可能相关的信息：

- 使用高亮或笔记记录
- 暂时不判断最终是否采用
- 保持开放心态，宁可多标记

### Step 3: 相关性筛选

对标记的信息进行筛选：

- 应用相关性判断框架
- 移除低相关性的内容
- 保留高和中相关性的内容

### Step 4: 分类和组织

将筛选后的信息按类别组织：

- 分配到相应的类别（NEW、BEST_PRACTICE 等）
- 合并相似的内容
- 建立逻辑顺序

### Step 5: 提炼和格式化

对每条信息进行提炼：

- 用简洁的语言重述
- 添加必要的上下文
- 按照统一格式整理
- 确保可操作性

### Step 6: 质量检查

使用质量评估标准进行检查：

- 完整性、准确性、可操作性
- 结构化、简洁性
- 修正发现的问题

### Step 7: 准备用户确认

将提取的信息整理成清晰的格式：

- 按类别分组展示
- 每条信息简明扼要
- 标注建议和理由
- 便于用户快速决策

## 实际示例

### 示例 1：从技术博客提取信息

**输入**：一篇关于 React Hooks 最佳实践的博客文章

**提取结果**：

```
[BEST_PRACTICE] useEffect 依赖数组管理
- 始终包含所有使用的变量
- 使用 ESLint 插件自动检查
- 避免使用空数组除非确实无依赖

[EDGE_CASE] useState 异步更新
- setState 是异步的，不要立即读取新值
- 需要使用 useEffect 或回调形式
- 示例: setCount(c => c + 1)

[GOTCHA] useCallback 过度使用
- 不是所有函数都需要 useCallback
- 仅在传递给子组件或作为依赖时使用
- 过早优化会增加复杂度

[EXAMPLE] 自定义 Hook 模式
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}
```

## 常见错误和避免方法

### 错误 1：提取过多背景信息

**问题**：包含了大量理论知识，但缺乏可操作性

**避免**：聚焦"如何做"而非"是什么"，保留必要的最小上下文

### 错误 2：忽略版本和依赖

**问题**：提取的信息没有标注适用的版本或前置条件

**避免**：始终标注版本要求、依赖项、兼容性信息

### 错误 3：复制粘贴而非提炼

**问题**：直接复制原文，没有进行提炼和重组

**避免**：用自己的话重述，提取核心要点，去除冗余

### 错误 4：分类不准确

**问题**：将最佳实践误标为新增知识，或反之

**避免**：先分析目标技能的现有内容，再确定分类

### 错误 5：缺乏上下文

**问题**：提取的信息孤立存在，不知道何时何地使用

**避免**：添加适用场景、前提条件、预期效果

## 工具辅助

虽然主要由人工判断，但可以借助：

- **关键词搜索**：快速定位相关章节
- **代码高亮**：识别代码示例
- **结构化解析**：利用 Markdown 标题层级
- **对比工具**：与现有内容对比避免重复

## 总结

信息提取是技能增强的关键第一步。遵循本指南的原则和方法，可以系统地识别和提取有价值的信息，为后续的整合工作奠定坚实基础。

记住核心要点：

1. 相关性优先，只提取对技能真正有用的信息
2. 结构化分类，便于后续整合
3. 保持可操作性，确保信息可以直接应用
4. 质量把关，通过评估标准确保提取质量
5. 准备确认，以清晰的格式呈现给用户
