# Enhance Skill - 技能增强助手

本技能用于帮助用户从各种输入资料（文档、代码、对话记录等）中提取有价值的信息，并通过交互式确认流程，智能地将这些信息整合优化到指定的现有技能包中。

## 前置条件

使用本技能前，确保：

- 目标技能包已存在且可以正常加载
- 用户提供了需要处理的输入资料（文本、代码、文档片段等）
- 明确要增强的具体技能名称

**如果是从零开始创建技能**，请先使用 create-skill 技能搭建基础框架。

## 使用流程

1. **确认目标技能**: 询问用户要增强哪个现有技能包
2. **评估技能能力**: 读取目标技能的 SKILL.md 和相关 references，理解其当前能力和范围
3. **接收输入资料**: 接受用户提供的各种类型资料
4. **智能提取信息**: 分析资料，识别与目标技能相关的有价值信息
5. **交互式确认**: 向用户展示提取的信息，让用户确认哪些需要添加
6. **智能整合优化**: 根据信息类型和用户确认，选择最佳方式整合到技能包中

## 关键能力摘要

### 信息提取

- 处理多种类型的输入资料（文档、代码、技术规范、对话记录等）
- 智能识别与目标技能相关的有价值信息
- 信息分类：新增知识、最佳实践、边界情况、示例代码、注意事项等
- 相关性判断和质量评估

### 技能能力分析

- 快速理解技能的核心能力和职责范围
- 识别技能的知识缺口和改进空间
- 评估新信息与现有内容的关系
- 判断信息价值和优先级

### 智能整合策略

- 决策是否更新现有文件还是创建新文件
- 优化 SKILL.md 的描述或指令
- 维护渐进式披露原则（SKILL.md < 500 行）
- 内容去重和冲突解决
- 保持技能职责清晰，避免功能膨胀

### 交互式确认流程

- 向用户清晰展示提取的信息
- 按类别组织待确认内容
- 允许用户选择性采纳
- 提供整合方案说明

## 常见应用场景

### 场景 1：基于新技术文档增强技能

**问题**：发布了新的 API 文档或技术指南，需要更新相关技能  
**解决方案**：

1. 确认目标技能（如 "api-integration" 技能）
2. 加载 references/skill-analysis.md 分析当前技能能力
3. 用户提供新技术文档
4. 加载 references/extraction-guide.md 提取相关信息
5. 向用户展示提取的新 API 用法、最佳实践等
6. 用户确认后，加载 references/integration-strategies.md 进行整合
7. 可能创建新的 reference 文件或更新现有文件

### 场景 2：从项目经验中提取最佳实践

**问题**：团队在实际使用中积累了经验，需要将这些经验固化到技能中  
**解决方案**：

1. 确认要增强的技能
2. 用户提供项目经验总结、问题记录等
3. 提取最佳实践、常见陷阱、解决方案等
4. 用户确认后整合到技能的 references 中
5. 可能在 SKILL.md 中添加 Gotchas 章节引用

### 场景 3：补充边界情况和注意事项

**问题**：技能在某些边界情况下表现不佳，需要补充相关知识  
**解决方案**：

1. 用户提供边界情况的描述和解决方案
2. 提取关键的边界条件和处理方法
3. 整合到现有 reference 文件或创建新的边界情况文档
4. 在 SKILL.md 中更新相关引用

### 场景 4：基于社区反馈改进技能

**问题**：收到用户对技能的反馈和建议  
**解决方案**：

1. 整理用户反馈内容
2. 提取有价值的改进建议
3. 分析对现有技能的影响
4. 实施相应的增强（可能是描述优化、指令改进、新增示例等）

## 工作流程示例

```text
用户: "我有一份新的 Python 异步编程最佳实践文档，想用来增强我的 async-coding 技能"

你:
1. 确认目标技能是 "async-coding"
2. 读取 skills/async-coding/SKILL.md 和相关 references
3. 加载 references/skill-analysis.md 分析当前技能能力
4. 接收用户提供的最佳实践文档
5. 加载 references/extraction-guide.md 提取相关信息
6. 向用户展示提取的内容：
   - 新的异步模式（3项）
   - 性能优化技巧（2项）
   - 常见陷阱（4项）
7. 用户确认需要添加所有内容
8. 加载 references/integration-strategies.md 制定整合方案
9. 执行整合：
   - 创建 references/advanced-patterns.md
   - 更新 references/performance-tips.md
   - 在 SKILL.md 中添加对新文件的引用
10. 生成增强报告供用户 review
11. 建议使用 optimize-skill 进行测试验证
```

## 与相关技能的关系

- **create-skill**：从零创建新技能的基础框架
- **optimize-skill**：优化现有技能的质量、描述准确性和触发率
- **enhance-skill**：从外部资料中提取信息并增强现有技能的内容

**典型工作流**：

1. 使用 create-skill 创建基础框架
2. 使用 enhance-skill 从外部资料增强内容
3. 使用 optimize-skill 优化描述和测试验证
4. 根据需要重复步骤 2-3 持续改进

三个技能协同工作，覆盖技能的完整生命周期。

## 重要提醒

- **始终进行交互式确认**：不要自动添加未经用户确认的内容
- **保持技能职责单一**：避免通过增强使技能承担过多职责
- **遵循渐进式披露**：详细内容放入 references/，保持 SKILL.md 简洁
- **注意内容去重**：避免添加与现有内容重复的信息
- **维护引用关系**：新增 reference 文件时，确保在 SKILL.md 中正确引用

记住：你的目标是帮助用户**系统化地增强技能内容**，使其更加全面和实用，同时保持清晰的结构和职责边界。
